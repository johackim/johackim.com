Modèles de prix pour un SaaS

Mis à jour le dimanche 27 juin 2021 par johackim

Il existe plusieurs modèles de prix pour un SaaS :

  • Flat-rate pricing (ex : basecamp)
  • Usage-based pricing (ex : Stripe)
  • Tier-based pricing (ex: Kinsta)
  • User-based pricing (ex : Calendly)
  • Feature-based pricing (ex : Intercom)
  • Credit-based pricing (ex : Audible)
  • Hybrid pricing (ex : Drift)

Flat-rate pricing

49€ par mois

Ex: basecamp.

Le modèle de prix Flat-rate est la façon la plus simple de vendre un produit.

Avec ce modèle, il existe seulement un produit avec un seul prix.

Il peut y avoir des variances selon si le prix est par mois ou pas année, mais l'utilisateur accède à toutes les fonctionnalités.

C'est plus facile à vendre, car il n'existe qu'un modèle de prix, ce qui génère moins de confusions.

C'est aussi un modèle qui a l'avantage d'être prédictible.

| Avantages | Inconvénients | | ------------------------- | ---------------- | | Facilité de communication | Inflexible | | Simple | Pas d'expansions | | Prévisible | | | Diversifié | |

Usage-based pricing

9€ pour 1,000 emails

Ex: Stripe, Postmark, Convertkit.

C'est un modèle pay-as-you-go.

Plus vous utiliser un service (ex: une api, un espace de stockage), plus vous payer. Et inversement, moins vous utiliser un service moins vous payer.

Vous pouvez être facturé par chaque envoie d'email, par un appel à une API, etc...

Ou être facturé sur un pourcentage de vos revenus.

| Avantages | Inconvénients | | --------------------------------- | --------------------------------- | | Facile à comprendre | Pas de lien direct avec la valeur | | Non-discriminatoire | Non prévisible | | Vous évoluez en même temps qu'eux | Encourage la frugalité |

Tier-based pricing

Ex: Kinsta, Fomo.

Starter (9€/mois) - Pro (49€/mois) - Enterprise (499€/mois)

Ce modèle présente différentes version du produit avec des fonctionnalités différentes pour un prix différent.

Le noms des différents tiers doit être claire et explicite pour éviter les confusions.

| Avantages | Inconvénients | | --------------------------------------- | ---------------------- | | Attrait pour plusieurs types de clients | Potentiellement confus | | Potentiel d'expansion | Compliqué | | Similaire à la tarification à l'usage | Analyses difficiles |

User-based pricing

9€ par utilisateur

Ex: Notion, Calendly, Close.

Les clients sont facturé par nombre d'utilisateurs.

Ce modèle doit uniquement être utilisé pour les produits avec une expérience unique qui varie selon la personne qui utilise le compte.

Sinon les utilisateurs vont probalement se partagé un seul identifiant, ce qui peut faire perdre un certain nombre de revenus.

| Avatanges | Invonvénients | | ------------------------------------ | ---------------------------------------------- | | Facile à comprendre | Décourage l'adoption | | Revenue scales with company adoption | L'utilisation par les utilisateurs peut varier | | Prévisible | Contraction des revenus |

Feature-based pricing

X (99€ / mois), Y (49€ / mois), Z (59€ / mois)

Ex: Intercom, baremetrics

| Avantages | Inconvénients | | ---------------------- | -------------------------------------------- | | Voie de l'expansion | Réticence à payer plus | | Focus sur le marketing | "Add-on" peut avoir une connotation négative | | Modularité | |

Credit-based pricing

$9 for 4 books a month

Ex: Audible

| Avantages | Inconvénients | | ------------------------- | -------------------------------------------- | | Facturé avant utilisation | Probabilité plus élevée de remboursement | | Simple | Surplus si on ne l'utilise pas régulièrement | | Décourage la frugalité | |

Hybrid pricing

Ex: Drift

Core 99€/mois + 9€ / user + Add-on 29€ / mois

Ex: Drift, Appcues.

C'est le modèle qui mixe tous les modèles de prix.

| Avantages | Inconvénients | | ---------------------------------- | ---------------------------------- | | Se différencier des concurrents | Devient rapidement complexe | | Possibilités d'expansion multiples | Peut être trop, pour trop de monde |
